# "AI 캡스톤디자인 프로젝트 기획서" 구성 제목과 작성 방법

## 작성 방법과 기한
- 적절한 파워포인트 형식으로 작성한다. 
- 팀 깃허브 저장소에 PDF 파일로 업로드한다. 
- 2026.5.5(화) 수업에서 업로드한 자료로 발표한다. 

## 핵심 요약
- 각 항목은 “무엇을 왜 어떻게 만들 것인가”를 명확히 드러내는 것이 핵심이다
- 단순 설명이 아니라 근거 + 구체성 + 실행 가능성을 포함해야 한다
- AI 프로젝트는 반드시 데이터·모델·서비스 흐름이 연결되어야 한다
- 평가자는 “실현 가능성”과 “문제 해결력”을 중심으로 판단한다

--- 
## 1. 프로젝트 개요
### 1.1 개발 팀명과 깃허브 저장소
- 팀명과 의미, 깃허브저장소  
    - 팀명은 프로젝트의 핵심 아이디어 또는 방향성을 반영하여 작성한다
    - 팀명 의미를 반드시 설명하고, 프로젝트 주제와의 연관성을 제시한다
    - GitHub 저장소 주소(URL)를 명시하고, 저장소 구조(코드/데이터/문서)를 간략히 설명한다
        - 예: “FoodAI팀 – 음식 이미지 분류 AI 개발을 목표로 구성된 팀”

### 1.2 팀 역할 분담
- 기획, 데이터, 모델링, UI/UX
    - 팀원별 역할을 기능 단위 기준으로 명확히 구분하여 작성한다
    - 최소 4가지 역할 포함: 기획, 데이터, 모델링, UI/UX
    - 각 역할에 대해 “구체적인 수행 업무”를 기술한다
        - 예: 데이터 담당 → 데이터 수집, 라벨링, 전처리 수행

### 1.3 프로젝트 제목
- 적합한 프로젝트 제목
    - 한 문장으로 프로젝트 핵심을 표현한다
    - “입력 데이터 + 처리 방식(AI) + 결과” 구조로 작성한다
    - 모호한 표현 대신 구체적인 기능이 드러나도록 작성한다
        - 예: “이미지 기반 음식 자동 분류 및 칼로리 예측 AI 시스템”

### 1.4 프로젝트 배경 및 필요성
- 사회적/기술적 배경, 문제 인식 및 추진 동기  
    - 사회적 문제 또는 기술적 한계를 기반으로 문제를 정의한다
    - 기존 방식의 문제점과 개선 필요성을 논리적으로 제시한다
    - “왜 이 프로젝트를 해야 하는가”를 중심으로 작성한다
        - 예: 음식 관리의 어려움 → 자동화 필요성

### 1.5 프로젝트 주요 기능 및 내용 요약
- 핵심 아이디어, 접근 방법, 개발 범위 요약  
    - 핵심 기능 3~5개를 중심으로 간결하게 정리한다
    - 입력 → 처리 → 출력 흐름으로 설명한다
    - 전체 시스템 구조를 한 문단 또는 항목으로 요약한다
        - 예: 이미지 입력 → CNN 모델 → 음식 분류 결과 출력

### 1.6 프로젝트 목표 및 기대효과
- 프로젝트가 달성하고자 하는 구체적 성과와 기대되는 변화  
    - 정량적 목표(정확도, 속도 등)를 포함하여 작성한다
    - 사용자 또는 사회에 미치는 영향을 구체적으로 기술한다
    - “성과 지표” 중심으로 작성하는 것이 중요하다

## 2. 프로젝트 개발을 위한 기술 적용 계획
### 2.1 시장 벤치마킹
- 유사 제품 앱·시스템 분석, 주요 화면 및 주요 기능 설명 
    - 유사 서비스 2~3개를 선정하여 분석한다 
    - 주요 기능, UI 구성, 장단점을 비교한다 
    - 본 프로젝트와의 차별성을 명확히 제시한다 

### 2.2 적용할 AI 기술 개요
- 생성형 AI, 딥러닝, 자연어 처리(NLP), 강화학습 등  
    - 사용할 AI 기술을 구체적으로 명시한다
        - 예: CNN, Transformer, GPT, RL 등
    - 각 기술의 역할과 적용 이유를 함께 설명한다

### 2.3 모델 구조 및 알고리즘 설계
- 데이터 흐름도, 학습 모델 개념도, 주요 알고리즘 설명  
    - 전체 데이터 흐름을 단계별로 설명한다
    - 입력 → 전처리 → 모델 → 출력 구조를 명확히 한다
    - 모델 구조(예: ResNet, LSTM 등)를 선택한 이유를 포함한다
    - 가능하면 간단한 다이어그램 또는 블록 구조로 표현한다

### 2.4 데이터 수집 및 전처리 계획
- 데이터 출처, 수집 방식, 전처리 방법, 윤리적 고려사항
    - 데이터 출처(공개 데이터, 직접 수집 등)를 명시한다
    - 데이터 수집 방법과 규모를 구체적으로 작성한다
    - 전처리 과정(정규화, 라벨링 등)을 단계적으로 설명한다
    - 개인정보 및 윤리적 문제를 반드시 고려한다

### 2.5 개발환경 및 구현 도구
- 개발언어 및 환경, 생성형AI, 프레임워크(PyTorch, TensorFlow, LangChain 등), 하드웨어 환경 
    - 개발 언어(Python 등)와 프레임워크를 명시한다
    - AI 도구(PyTorch, TensorFlow 등)와 사용 이유를 설명한다
    - 하드웨어(GPU, 클라우드 등) 환경을 포함한다

## 3. 프로젝트 추진 계획
### 3.1 프로젝트 일정표 (Gantt 차트)
- 주차별 활동 계획, milestone 설정
    - 주차별 주요 작업을 단계적으로 나열한다
    - 기획 → 데이터 → 모델 → 구현 → 테스트 흐름으로 구성한다
    - 각 단계별 milestone을 명확히 설정한다

### 3.2 프로토타입 개발 계획
- 1학기말 발표를 위한 웹, 앱, 시스템 개발 계획
    - 1차 결과물의 형태(웹/앱/시스템)를 명시한다
    - 핵심 기능 중심으로 구현 범위를 제한하여 작성한다
    - 데모 시연이 가능한 수준까지 목표를 설정한다

### 3.3 사용자 테스트 또는 설문 계획
- 사용성 평가(UX), 만족도 조사 방법  
    - 테스트 대상(학생, 일반 사용자 등)을 정의한다
    - 평가 항목(사용성, 만족도 등)을 구체화한다
    - 설문 또는 실험 방법을 명확히 기술한다

### 3.4 결과 분석 및 개선 방안 제시
- 피드백을 반영한 모델 개선 또는 기능 보완 계획  
    - 수집된 데이터를 기반으로 분석 방법을 제시한다
    - 문제점 도출 → 개선 전략 수립 구조로 작성한다
    - 모델 개선(튜닝) 또는 기능 확장 계획을 포함한다

---
## 4. 기대효과 및 향후 발전 방향
### 4.1 기대효과 요약
- 프로젝트가 가져올 산업적/교육적/사회적 가치  
    - 산업적, 교육적, 사회적 효과를 구분하여 작성한다
    - 실제 활용 가능성과 영향력을 중심으로 설명한다

### 4.2 한계점 및 개선 가능성
- 현재 기술적·운영적 제약사항 및 해결 가능성  
    - 데이터 부족, 모델 성능 한계 등 현실적 제약을 기술한다
    - 각 한계에 대한 해결 가능성 또는 대응 전략을 제시한다

### 4.3 후속 연구 또는 서비스 확장 방향
- 상용화, 고도화, 지속적 발전 방향 제시  
    - 향후 기능 확장 또는 상용화 가능성을 제시한다
    - 추가 연구 주제 또는 고도화 방향을 포함한다

## 5. 참고 문헌 및 자료
- 참고한 논문, 데이터셋, 프레임워크 공식 문서, 기사 등 명시  
    - 참고 문헌 작성 방법
        - 논문, 데이터셋, 공식 문서를 정확한 형식으로 작성한다
        - 최소 3개 이상 포함하는 것이 바람직하다
        - 예: 저자, 제목, 학회/출판사, 연도 순으로 기술한다
